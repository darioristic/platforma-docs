# Get Order

<ApiEndpoint method="GET" path="/v1/orders/:id" auth="Bearer token">
  Retrieves a single order by its unique identifier, including provisioning status.
</ApiEndpoint>

## Path Parameters

<ParamTable>
  <Param name="id" type="string" required>
    The unique order identifier (e.g., `ord_xyz789`).
  </Param>
</ParamTable>

## Order Statuses

| Status | Description |
|--------|-------------|
| `pending` | Order created, awaiting processing |
| `processing` | Order is being validated and provisioned |
| `provisioned` | All resources have been provisioned successfully |
| `partially_provisioned` | Some items provisioned, others still pending |
| `failed` | Provisioning failed — check resource statuses for details |
| `cancelled` | Order was cancelled before completion |

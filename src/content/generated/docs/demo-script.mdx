# Cloud Factory — Demo Script

**Duration:** ~20 minutes
**Audience:** Potential client / T-2 management
**Goal:** Demonstrate the complete flow from product browsing to an active cloud service

---

## Preparation

```bash
# Start infrastructure and services
docker compose -f docker-compose.dev.yml up -d
npm run dev

# Seed demo data (if not already done)
npm run seed
```

Verify that all services are healthy:

- Admin Platform: &lt;http://localhost:3100>
- Customer Portal: &lt;http://localhost:3200>
- Public Web: &lt;http://localhost:3000>

## Scenario 3: New Order (End-to-End Flow)

**Goal:** Demonstrate the complete flow from order to active service

1. **Open Public Web** (&lt;http://localhost:3000>)
   - Show landing page (professional design, 14 sections)
   - Scroll to Products section

2. **Browse Products**
   - Click on VPS category
   - Show specifications and prices (EUR)
   - Add "Business VPS 8" to cart

3. **Checkout**
   - Open cart
   - Review items and total amount
   - Click "Proceed to Checkout"
   - Fill in billing information
   - Confirm order

4. **Track Provisioning**
   - Deploy page displays a 6-step progress bar
   - Status progresses: DRAFT -> PENDING_PAYMENT -> PAYMENT_CONFIRMED -> PROVISIONING
   - Provisioning Worker automatically processes the request (4 steps)
   - Upon completion: status FULFILLED

5. **Verify in Admin Platform**
   - Switch to Admin (&lt;http://localhost:3100>)
   - Show the new order in the Orders table
   - Show the provisioning request in the Provisioning section
   - Show the new invoice in the Billing section
   - Show the notification in the bell icon

## Scenario 4: Technical Capabilities

**Goal:** Show the technical maturity of the platform

1. **API Documentation**
   - Open Swagger: &lt;http://localhost:3003/api> (Identity Service)
   - Show authentication, users, API keys
   - Open &lt;http://localhost:3002/api> (Order Service)
   - Show order lifecycle endpoints

2. **Distributed Tracing**
   - Open Jaeger: &lt;http://localhost:16686>
   - Search for traces for "order-service"
   - Show spans that pass through multiple services

3. **Event-Driven Architecture**
   - Explain the Kafka event flow:
     - Order created -> Provisioning triggered -> Billing notified
   - Saga pattern for distributed transactions
   - Dead Letter Queue for failed events

4. **Security**
   - JWT authentication with refresh tokens
   - Role-Based Access Control (RBAC)
   - Rate limiting (nginx + application level)
   - Helmet security headers
   - CORS validation
   - Swagger disabled in production

## Key Messages for the Client

1. **Sovereign cloud** — The platform is fully operated in Slovenia
2. **Enterprise-grade** — 7 microservices, event-driven, saga orchestration
3. **Self-service** — Customers can independently manage their services
4. **Observability** — Tracing, metrics, health monitoring built in
5. **Extensibility** — Modular architecture, easy to add new cloud providers
6. **Automation** — From order to active service without manual intervention

## Known Limitations (MVP)

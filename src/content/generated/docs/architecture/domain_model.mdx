## 2. Product Domain

### 2.1 Bounded Context

The Product domain owns all aspects of service definitions, pricing, and catalog management.

### 2.2 Aggregates

#### Product Aggregate

```typescript
// Aggregate Root
interface Product {
  id: ProductId;
  tenantId: TenantId;
  code: string;                    // Unique product code
  name: LocalizedString;
  description: LocalizedString;
  category: ProductCategory;
  type: ProductType;
  status: ProductStatus;
  specifications: ProductSpecification[];
  pricingRules: PricingRuleId[];

  | 'VPS'
  | 'KUBERNETES_CLUSTER'

  | 'ADDON';

  | 'DRAFT'
  | 'ACTIVE'
  | 'DEPRECATED'

  key: string;
```

### 3.3 Order State Machine

```
     ┌─────────┐
     │  DRAFT  │
     └────┬────┘
          │ submit()
          ▼
┌─────────────────┐
│ PENDING_PAYMENT │◄───────────────────────┐
└────────┬────────┘                        │
         │ paymentConfirmed()              │ paymentFailed()
         ▼                                 │
┌─────────────────┐                        │
│PAYMENT_CONFIRMED│────────────────────────┤
└────────┬────────┘                        │
         │ startValidation()               │
         ▼                                 │
   ┌───────────┐                           │
   │ VALIDATING│───────────────────────────┤
   └─────┬─────┘    validationFailed()     │
         │ validationPassed()              │
         ▼                                 │
  ┌─────────────┐                          │
  │PROVISIONING │──────────────────────────┤
  └──────┬──────┘   provisioningFailed()   │
         │                                 │
         ├── itemProvisioned() ──┐         │
         │                       ▼         │
         │            ┌──────────────────┐ │
         │            │PARTIALLY_FULFILLED│ │
         │            └────────┬─────────┘ │
         │                     │           │
         │◄────────────────────┘           │
         │ allItemsProvisioned()           │
         ▼                                 │
   ┌───────────┐                    ┌──────┴──────┐
   │ FULFILLED │                    │   FAILED    │
   └───────────┘                    └─────────────┘
         │
         │ cancel() (within window)
         ▼
   ┌───────────┐
   │ CANCELLED │
   └───────────┘
```

### 7.2 Anti-Corruption Layers

| Boundary | ACL Location | Purpose |
|----------|--------------|---------|
| Product → Order | Order Service | Translate product specs to order items |
| Order → Provisioning | Provisioning Service | Translate orders to infra requests |
| Billing Integration → Central BSS | Billing Adapter | Protocol & model translation |
| Identity → Corporate IdP | IAM Service | SAML/OIDC federation |

### 8.2 Event Schema Versioning

All events follow semantic versioning:
- Event type includes version: `cloud-factory.order.created.v1`
- Schema registry enforces compatibility
- Breaking changes require new version

### 8.3 Event Retention

| Namespace | Retention | Compaction |
|-----------|-----------|------------|
| audit.* | Indefinite | None |
| order.* | 2 years | After 90 days |
| provisioning.* | 1 year | After 30 days |

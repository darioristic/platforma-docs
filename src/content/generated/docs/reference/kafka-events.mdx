# Kafka Events Reference

All Cloud Factory services communicate asynchronously via Apache Kafka 7.5. Events follow a standardized envelope format with topic naming conventions.

## Event Envelope

Every event published to Kafka wraps its domain-specific payload in a `CloudFactoryEvent&lt;T>` envelope:

```typescript
interface CloudFactoryEvent<T> {
  eventId: string; // UUID v7
  eventType: string; // domain.action (e.g. "order.created")
  version: string; // Schema version (e.g. "v1")
  timestamp: string; // ISO 8601
  tenantId: string; // Tenant isolation
  correlationId: string; // Request tracing
  causationId?: string; // ID of the event that caused this
  actor: EventActor; // Who triggered the event
  payload: T; // Domain-specific data
  metadata?: Record<string, unknown>;
}

interface EventActor {
  id: string; // User ID, service name, etc.
  type: 'user' | 'service' | 'system' | 'api_key';
}
```

## Topic Naming Convention

```
cloud-factory.{env}.{domain}.{event}.v{version}
```

- **env**: `dev`, `staging`, `prod`, `test` (prevents cross-environment leakage)
- **domain**: `order`, `product`, `provisioning`, `billing`, `identity`, `notification`, `support`, `reporting`, `dns`, `audit`
- **event**: kebab-case event name
- **version**: schema version (currently `v1`)

Example: `cloud-factory.prod.order.created.v1`

## Dead Letter Queue (DLQ)

Failed messages are routed to `\{original-topic\}.dlq` after exhausting retries. The DLQ handler service tracks failure reasons and supports replay.

## Event Flow: Order Lifecycle (Saga)

```
1. order.created           -> Notification: "Order received" email
2. order.payment-confirmed -> Order Service starts provisioning
3. order.provisioning-started -> Provisioning Service creates resources
4. provisioning.started    -> Progress tracking
5. provisioning.step-completed (x N)
6. provisioning.completed  -> Order Service marks item fulfilled
7. order.service-delivered  -> Notification: delivery email with credentials
8. order.fulfilled         -> Billing: create subscription + invoice
                           -> Notification: "Order complete" email
```

On failure at any step, the saga triggers compensation:

```
order.compensation-started -> rollback provisioned resources, issue refund
order.compensation-completed | order.compensation-failed
```

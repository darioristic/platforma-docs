# Database Schema Reference

Cloud Factory uses PostgreSQL 16 for six services (each with its own schema) and MongoDB 7 for the Product Service catalog.

## PostgreSQL Schemas

### Identity Schema (`identity`)

#### `identity.users`

| Column           | Type         | Constraints             | Description                                   |
| ---------------- | ------------ | ----------------------- | --------------------------------------------- |
| id               | uuid         | PK, generated           | User identifier                               |
| email            | varchar      | unique                  | Login email                                   |
| firstName        | varchar      |                         | First name                                    |
| lastName         | varchar      |                         | Last name                                     |
| passwordHash     | varchar      | select: false           | Bcrypt hash (excluded from queries)           |
| roles            | simple-array |                         | Array of role names                           |
| tenantId         | uuid         | FK -> tenants           | Owning tenant                                 |
| status           | varchar      | default 'active'        | `active`, `suspended`, `pending_verification` |
| mfaEnabled       | boolean      | default false           | Whether MFA is active                         |
| mfaSecret        | varchar      | nullable, select: false | TOTP secret (encrypted)                       |
| mfaBackupCodes   | simple-array | nullable, select: false | One-time backup codes                         |
| lastLoginAt      | timestamp    | nullable                | Last successful login                         |
| lastLogoutAt     | timestamp    | nullable                | Last logout                                   |
| loginAttempts    | int          | default 0               | Failed login counter                          |
| lockedUntil      | timestamp    | nullable                | Account lock expiry                           |
| refreshTokenHash | varchar      | nullable, select: false | Hashed refresh token                          |
| erasedAt         | timestamp    | nullable                | GDPR erasure timestamp                        |
| createdAt        | timestamp    | auto                    | Creation timestamp                            |
| updatedAt        | timestamp    | auto                    | Last update                                   |

**Relationships:** ManyToOne -> `tenants`, OneToMany -> `oauth_accounts`

#### `identity.tenants`

| Column    | Type       | Constraints   | Description                                                             |
| --------- | ---------- | ------------- | ----------------------------------------------------------------------- |
| id        | uuid       | PK, generated | Tenant identifier                                                       |
| name      | varchar    | unique        | Organization name                                                       |
| type      | varchar    |               | `customer`, `partner`, `internal`                                       |
| country   | varchar(2) |               | ISO 3166-1 alpha-2 code                                                 |
| settings  | jsonb      | nullable      | TenantSettings (locale, timezone, currency, limits, features, branding) |
| config    | jsonb      | nullable      | Arbitrary configuration                                                 |
| createdAt | timestamp  | auto          |                                                                         |
| updatedAt | timestamp  | auto          |                                                                         |

**Relationships:** OneToMany -> `users`

**TenantSettings shape:**

```json
{
  "locale": "en",
  "timezone": "Europe/Belgrade",
  "currency": "EUR",
  "dateFormat": "DD.MM.YYYY",
  "maxUsers": 50,
  "maxInstances": 10,
  "features": {
    "supportEnabled": true,
    "billingEnabled": true,
    "provisioningEnabled": true,
    "mfaRequired": false
  },
  "rateLimitTier": "basic",
  "branding": { "logoUrl": "", "primaryColor": "", "companyUrl": "" }
}
```

### Orders Schema (`orders`)

#### `orders.orders`

| Column      | Type      | Constraints           | Description                                                                                                                              |
| ----------- | --------- | --------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| id          | uuid      | PK                    |                                                                                                                                          |

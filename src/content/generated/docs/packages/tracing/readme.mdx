# @cloud-factory/tracing

OpenTelemetry tracing setup with OTLP export, Pino log correlation, and Kafka trace propagation.

## Installation

```json
"@cloud-factory/tracing": "*"
```

## Key Exports

| Export                         | Description                                                             |
| ------------------------------ | ----------------------------------------------------------------------- |
| `initTracing(config)`          | Initialize the OpenTelemetry NodeSDK with auto-instrumentation          |
| `getTracingSdk()`              | Retrieve the running SDK instance                                       |
| `TracingConfig`                | Config type: `serviceName`, `serviceVersion`, `otlpEndpoint`, `enabled` |
| `pinoTraceMixin()`             | Pino mixin that injects `trace_id` and `span_id` into log lines         |
| `injectTraceHeaders(headers?)` | Add W3C trace context headers to outgoing requests                      |
| `extractTraceContext(headers)` | Extract trace context from incoming Kafka message headers               |

## Usage

### Bootstrap tracing (call before NestJS init)

```typescript
import { initTracing } from '@cloud-factory/tracing';

initTracing({
  serviceName: 'order-service',
  otlpEndpoint: 'http://jaeger:4318/v1/traces',
});
```

### Correlate logs with traces

```typescript
import { pinoTraceMixin } from '@cloud-factory/tracing';

LoggerModule.forRoot({
  pinoHttp: {
    mixin: pinoTraceMixin,
  },
});
```

### Propagate traces through Kafka

```typescript
import { injectTraceHeaders, extractTraceContext } from '@cloud-factory/tracing';

// Producer: attach trace headers to message
const headers = injectTraceHeaders();
await producer.send({ topic, messages: [{ value, headers }] });

// Consumer: restore trace context from message
const ctx = extractTraceContext(message.headers);
context.with(ctx, () => processMessage(message));
```

## Environment Variables

| Variable                      | Default                           | Description                  |
| ----------------------------- | --------------------------------- | ---------------------------- |
| `OTEL_TRACES_ENABLED`         | `true`                            | Disable tracing with `false` |
| `OTEL_EXPORTER_OTLP_ENDPOINT` | `http://localhost:4318/v1/traces` | OTLP collector URL           |
| `OTEL_SERVICE_VERSION`        | `0.1.0`                           | Reported service version     |

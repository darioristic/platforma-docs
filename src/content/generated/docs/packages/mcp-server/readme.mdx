# @cloud-factory/mcp-server

MCP (Model Context Protocol) server for Cloud Factory Admin Platform. Enables AI assistants (Claude Desktop, Cursor, Claude Code) to interact with the platform — query KPIs, manage orders, review invoices, and handle support tickets.

## Quick Start

```bash
# Build
npm run build -w @cloud-factory/mcp-server

# Run (requires backend services)
CF_ADMIN_EMAIL=admin@example.com CF_ADMIN_PASSWORD=secret node packages/mcp-server/dist/bin.js
```

## Architecture

```
src/
├── bin.ts                  # Entry point (stdio transport)
├── server.ts               # McpServer setup + registration
├── config.ts               # Env var parsing, service URL defaults
├── auth/
│   └── auth-manager.ts     # Lazy login, token cache, auto-refresh
├── http/
│   └── api-client.ts       # Axios client with auth interceptor
├── tools/
│   ├── index.ts            # registerAllTools()
│   ├── reporting.ts        # 5 reporting tools
│   ├── identity.ts         # 5 identity tools
│   ├── orders.ts           # 4 order tools
│   ├── products.ts         # 3 product tools
│   ├── billing.ts          # 4 billing tools
│   ├── provisioning.ts     # 3 provisioning tools
│   └── support.ts          # 4 support tools
└── resources/
    └── index.ts            # 5 MCP resources
```

**Auth flow:** Lazy authentication — login happens on first tool call, not on server startup. Tokens are cached in memory and auto-refreshed via axios interceptor on 401 responses.

**Transport:** Stdio only (stdin/stdout JSON-RPC). The server runs as a local process spawned by the MCP client.

# @cloud-factory/idempotency

NestJS module for request idempotency using the `Idempotency-Key` HTTP header. Stores responses in PostgreSQL via TypeORM and replays them on duplicate requests.

## Installation

```json
"@cloud-factory/idempotency": "*"
```

## Key Exports

- **`IdempotencyModule`** -- Dynamic module with `.forRoot()` (global interceptor) and `.forFeature()` (manual).
- **`@Idempotent(ttlMs?)`** -- Decorator to mark controller methods as idempotent (default TTL: 24h).
- **`IdempotencyService`** -- Low-level service for storing/retrieving idempotency keys.
- **`IdempotencyInterceptor`** -- NestJS interceptor that checks/stores responses.
- **`IdempotencyKey`** -- TypeORM entity for the `idempotency_keys` table.

## Usage

### Register in AppModule

```typescript
import { IdempotencyModule } from '@cloud-factory/idempotency';

@Module({
  imports: [
    IdempotencyModule.forRoot(), // registers global interceptor
  ],
})
export class AppModule {}
```

### Mark endpoints as idempotent

```typescript
import { Idempotent } from '@cloud-factory/idempotency';

@Controller('orders')
export class OrderController {
  @Post()
  @Idempotent() // uses default 24h TTL
  create(@Body() dto: CreateOrderDto) {
    return this.orderService.create(dto);
  }

  @Post('payment')
  @Idempotent(3600_000) // 1 hour TTL
  processPayment(@Body() dto: PaymentDto) {
    return this.paymentService.process(dto);
  }
}
```

Clients must include an `Idempotency-Key` header. Duplicate requests within the TTL return the cached response.

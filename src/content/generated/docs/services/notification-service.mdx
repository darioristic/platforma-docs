# Notification Service

**Port:** 3006 | **Database:** PostgreSQL | **Events:** Kafka (consumer + producer) | **WebSocket:** Socket.IO | **Email:** Resend

Central notification hub — receives events from all services and delivers notifications via email, in-app messages, and WebSocket. Supports user preferences, email templates with Handlebars, and per-tenant branding.

## 4. WebSocket Gateway

**Namespace:** `/notifications`
**CORS:** from CORS_ORIGIN env var
**Auth:** JWT token from `auth.token` or `Authorization` header

### Connection

1. Client connects with JWT
2. Token verified, userId extracted from `sub` claim
3. Client joined to room `user:\{userId\}`

### Server → Client Events

| Event                 | Description                          |
| --------------------- | ------------------------------------ |
| `notification`        | Real-time notification for user      |
| `provisioning-update` | Broadcast to all connections (admin) |

### Methods

- `sendToUser(userId, notification)` — Sends to all user connections
- `broadcastToAll(event, data)` — Broadcast to all
- `isUserOnline(userId)` — Check if user is online

## 6. Kafka Events

### Consumed Topics (21)

| Source           | Topics                                                                                               |
| ---------------- | ---------------------------------------------------------------------------------------------------- |
| **Order**        | created, fulfilled, failed, cancelled, service-delivered                                             |
| **Provisioning** | started, step-completed, completed, failed, rolled-back, service-suspended                           |
| **Billing**      | invoice-generated, payment-received, subscription-past-due, refund-completed, subscription-cancelled |
| **Support**      | ticket-created, comment-added, ticket-resolved, ticket-assigned, ticket-closed                       |
| **Identity**     | user-created                                                                                         |

### Produced Topics

| Topic                    | When                                |
| ------------------------ | ----------------------------------- |
| `notification.delivered` | Notification successfully delivered |
| `notification.failed`    | Delivery failed                     |

### Event Processing Logic

1. Consumer receives event from Kafka
2. Handler extracts tenantId, userId, email
3. Checks user preferences (PreferencesService)
4. Creates in-app notification if `inAppEnabled` && sends via WebSocket
5. Sends email if `emailEnabled` && email is present
6. Saves metadata (sanitized for user)

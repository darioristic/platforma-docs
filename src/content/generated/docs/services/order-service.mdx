# Order Service

**Port:** 3002 | **Database:** PostgreSQL | **Events:** Kafka (consumer + producer) | **Pattern:** Saga (fulfillment)

Manages the entire order lifecycle: from creation, through payment, infrastructure provisioning, to service delivery. Implements the saga pattern for orchestration and compensation.

## 2. State Machine

```text
  DRAFT ──► PENDING_PAYMENT ──► PAYMENT_CONFIRMED ──► VALIDATING ──► PROVISIONING
                                       │                                    │
                                       │              ┌─────────────────────┤
                                       │              │                     │
                                       │    PARTIALLY_FULFILLED      FULFILLED
                                       │              │                │
                                       └──────────────┴────────────────┘
                                                      │
                                              CANCELLED / FAILED
```

**Transitions:**

- `DRAFT → PENDING_PAYMENT`: submit()
- `PENDING_PAYMENT → PAYMENT_CONFIRMED`: confirmPayment()
- `PAYMENT_CONFIRMED → VALIDATING → PROVISIONING`: startProvisioning()
- `PROVISIONING → PARTIALLY_FULFILLED → FULFILLED`: provisioning events
- `Any non-terminal → CANCELLED`: cancel()
- `Any active → FAILED`: provisioning failure / saga compensation

**Terminal states:** FULFILLED, CANCELLED, FAILED

## 5. Saga Pattern (Fulfillment)

### Steps

1. **VALIDATE_ORDER** — Verify order data
2. **START_PROVISIONING** — Start infrastructure provisioning
3. **PROVISION*ITEM*\*** — Dynamic steps for each item

### Compensation (Rollback)

When provisioning fails (non-retryable or max retries):

1. Creates a compensation stack:
   - `CANCEL_PROVISIONING` for each provisioned item
   - `REFUND_PAYMENT` if payment was confirmed
2. Executes each compensation step sequentially (max 5 retries)
3. Publishes `order.compensation-started/completed/failed` events
4. Marks the order as FAILED

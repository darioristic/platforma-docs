# Support Service

**Port:** 3007 | **Database:** PostgreSQL | **Events:** Kafka (producer) | **SLA:** Cron-based monitoring

Manages support tickets, comments, SLA tracking, and breach detection. Supports priorities, categories, agent assignment, and internal communication.

## 2. API Endpoints

### Tickets Controller (`/tickets`)

| Method  | Route                 | Description                        | Guard                                      |
| ------- | --------------------- | ---------------------------------- | ------------------------------------------ |
| `POST`  | `/tickets`            | Create ticket                      | JWT, @Idempotent                           |

| `PATCH` | `/tickets/:id/reopen` | Reopen ticket                      | JWT                                        |

### Comments Controller (`/tickets/:ticketId/comments`)

| Method | Route                         | Description                            | Guard |
| ------ | ----------------------------- | -------------------------------------- | ----- |
| `POST` | `/tickets/:ticketId/comments` | Add comment                            | JWT   |

## 4. SLA Configuration

| Priority     | Response Time | Resolution Time |
| ------------ | ------------- | --------------- |
| **CRITICAL** | 15 min        | 4 hours         |
| **HIGH**     | 1 hour        | 8 hours         |
| **MEDIUM**   | 4 hours       | 24 hours        |
| **LOW**      | 8 hours       | 72 hours        |

### SLA Scheduler

- **Cron:** Every 5 minutes
- Checks breaches for response and resolution SLA
- Sets `responseBreached` / `resolutionBreached` flags
- Logs breaches

### First Response Tracking

- Recorded when the first agent/system comment is added
- Used to verify whether the response SLA has been met

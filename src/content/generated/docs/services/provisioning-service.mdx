# Provisioning Service

**Port:** 3004 | **Database:** PostgreSQL | **Events:** Kafka (consumer + producer) | **WebSocket:** Socket.IO (SSH console)

Manages the entire lifecycle of infrastructure resources: from request creation, through allocation and configuration, to verification and service activation. Supports OpenStack, OpenShift, and simulation providers.

## 2. State Machine

```text
  QUEUED ──► ALLOCATING ──► CONFIGURING ──► ACTIVATING ──► VERIFYING ──► COMPLETED
                │                │               │              │             │
                └────────────────┴───────────────┴──────────────┘             │
                                 │ (failure)                                  │
                                 ▼                                            │
                              FAILED ──► ROLLING_BACK ──► ROLLED_BACK        │
                                │                                             │
                                │ (retry, backoff)                            │
                                └──► back to failed step                     │
                                                                             │
                                                          COMPLETED ◄──► SUSPENDED
```

**Transitions:**

- `QUEUED → ALLOCATING`: startProvisioning()
- `ALLOCATING → CONFIGURING → ACTIVATING → VERIFYING → COMPLETED`: advanceStep()
- `Any active → FAILED`: failRequest() (increments retryCount)
- `FAILED → active step`: retryRequest() (if canRetry())
- `FAILED → ROLLING_BACK → ROLLED_BACK`: automatic rollback when retries are exhausted
- `COMPLETED ↔ SUSPENDED`: suspend/reactivate

## 5. Cloud Providers

### Provider Registry

Maps `ResourceType` to a specific provider via env variables:

```bash
PROVIDER_DEFAULT=simulation
PROVIDER_VPS=openstack
PROVIDER_KUBERNETES_CLUSTER=openshift
```

### SimulationProvider

- Default provider for development/testing
- 4-step pipeline with configurable delays
- Configurable failure rate (`SIMULATION_FAILURE_RATE`, default 0.1)
- Generates fake allocations (IP, hostname, endpoints)

### OpenStackProvider

- REST API integration (no SDK)
- Drivers: **Nova** (compute), **Neutron** (networking), **Cinder** (storage), **Ironic** (bare metal)
- Token refresh with TTL tracking
- Correlation ID propagation (`X-Correlation-Id` header)
- Real-time metrics (CPU, memory, disk, network) via Nova diagnostics

### OpenShiftProvider

- Kubernetes REST API
- Creates: Namespace + ResourceQuota + LimitRange + RoleBinding + NetworkPolicy
- Correlation ID propagation

## 6. Background Worker

`ProvisioningWorkerService` — polling-based worker that automatically processes requests.

### Configuration

| Env Variable                  | Default | Description                               |
| ----------------------------- | ------- | ----------------------------------------- |
| `PROVISIONING_WORKER_ENABLED` | true    | Enable/disable worker                     |
| `PROVISIONING_WORKER_POLL_MS` | 5000    | Interval between polls                    |
| `CB_FAILURE_THRESHOLD`        | 5       | Circuit breaker: failures before opening  |
| `CB_SUCCESS_THRESHOLD`        | 3       | Circuit breaker: successes before closing |
| `CB_OPEN_DURATION_MS`         | 60000   | Circuit breaker: open state duration      |
| `BULKHEAD_MAX_CONCURRENT`     | 3       | Bulkhead: max concurrent requests         |
| `BULKHEAD_MAX_QUEUE`          | 10      | Bulkhead: max queue size                  |
| `RETRY_BACKOFF_BASE_MS`       | 5000    | Backoff base (5s)                         |
| `RETRY_BACKOFF_MAX_MS`        | 300000  | Backoff cap (5min)                        |
| `STUCK_REQUEST_TIMEOUT_MS`    | 1800000 | Stuck detection (30min)                   |

### Features

1. **Automatic step processing** — Delegates to the appropriate provider for each step
2. **Exponential backoff retry** — Formula: `base * 2^(retryCount-1)`, capped at max
3. **Stuck request detection** — A step running longer than the timeout is automatically failed
4. **Automatic rollback** — When retries are exhausted and resources are allocated → deprovision
5. **Circuit breaker + Bulkhead** — Resilience wrapper for each step execution
6. **Graceful shutdown** — Waits up to 30s for in-progress requests to complete

### Worker Status (admin endpoint)

```json
{
  "enabled": true,
  "running": true,
  "processingCount": 2,
  "processingIds": ["req-abc", "req-def"],
  "pollIntervalMs": 5000,
  "circuitBreakerAvailable": true
}
```
